# ClawPet 玩法手册

这份文档不讲实现细节，只讲一件事：

现在这版 ClawPet，应该怎么玩。

如果你把它理解成一个“桌面像素宠物 + OpenClaw 交接界面”，这份手册就是它当前最合适的打开方式。

## 一句话理解

ClawPet 现在不是普通桌宠。

它已经可以作为：

- 你的桌面陪伴宠物
- OpenClaw 的实时状态指示器
- OpenClaw 的超轻交接入口
- 审批请求的桌面接球器

你可以把它理解成：

“一个会趴在桌面边上、会替 OpenClaw 跑来跑去、并在关键时刻提醒你接手的像素搭子。”

## 启动方式

在项目根目录运行：

```bash
npm run dev
```

如果你只是想看打包后的构建是否正常：

```bash
npm run build
npm run preview
```

## 启动后你会看到什么

启动后，桌面右下角会出现一只像素猫。

如果本机 OpenClaw / QClaw 正在运行，并且存在这些配置之一：

- `~/.openclaw/openclaw.json`
- `~/.qclaw/openclaw.json`

ClawPet 会自动尝试连接本机 Gateway，并开始同步：

- 当前会话
- 最近对话
- 在线 presence
- 节点列表
- 审批请求
- 当前运行中的任务活动

## 当前最推荐的 5 种玩法

## 1. 当作普通桌宠养着

最轻玩法，不需要 OpenClaw。

你可以：

- 单击宠物，让它打招呼
- 从托盘里添加更多猫
- 切换 `点击穿透`
- 切换 `暂停动作`

适合场景：

- 你只是想先看桌宠的整体感觉
- 你还没准备好接 OpenClaw
- 你想先确认这只宠物会不会打扰你的桌面操作

## 2. 当作 OpenClaw 状态灯

这是最自然的玩法之一。

当 OpenClaw 在不同状态下工作时，宠物会表现出不同的气质：

- `idle`：更松弛、更容易休息
- `thinking`：像在踱步思考
- `coding`：会更积极、更像在巡逻
- `running`：明显更活跃
- `waiting`：像在等你接手
- `error`：提醒感更强

而且这不只是粗粒度状态。

如果 Gateway 有更细的活动流，宠物还会尝试区分：

- `read`
- `write`
- `edit`
- `exec`
- `tool`
- `job`

所以你不打开任何面板，也能从桌面角落大概感知：

“它现在是在看东西、改东西，还是在跑东西。”

## 3. 当作 OpenClaw 的交接面板

双击宠物，会打开 OpenClaw 面板。

这是我目前最推荐的主玩法。

你可以在面板里做这些事：

- 看当前连接状态
- 看会话列表
- 切换当前 session
- 在像素衣橱里给全部宠物换皮
- 看最近对话
- 给当前 session 发一句话
- 停掉当前 run
- 查看待审批请求

它适合这种节奏：

1. 平时让宠物在桌面待着
2. 发现它开始忙起来
3. 双击打开面板
4. 看它在做哪条 session
5. 手动补一句指令或中断一下

这个交互比“每次都打开完整控制台”轻很多，也更像桌宠该有的感觉。

## 4. 当作审批接球器

这是这版最有价值的功能之一。

如果 OpenClaw 触发了 `exec.approval.requested`：

- 桌宠会进入更明显的提醒态
- 会自动弹出一个审批浮窗
- 你可以直接选：
  - `允许一次`
  - `总是允许`
  - `拒绝`
  - `打开面板`

这个玩法非常适合你在工作时不想一直盯着 Control UI 的场景。

它的意义是：

OpenClaw 不再只是“在后台跑”，而是会通过桌宠把需要你接手的时刻递到你面前。

## 5. 任务反馈与完成提醒

这是 P0-5 新增的功能，为 agent 工作形成完整的闭环反馈。

### 完成反馈

当 OpenClaw 中的任务完成时，宠物会：
- 显示庆祝文案（随机从"完成了！"、"搞定了！"等中选择）
- 产生闪烁的粒子效果

### 失败反馈

当 OpenClaw 中的任务失败时，宠物会：
- 显示温和的提示文案（随机从"出问题了..."、"嗯，失败了"等中选择）
- 帮助你理解出问题了，但不会让你感到焦虑

### 长时间等待提醒

如果任务在"等待审批"或"等待反馈"状态超过 30 秒，宠物会：
- 显示轻轻的催促文案（随机从"嗯？还在等呢..."、"怎么还没动静..."等中选择）
- 提醒你可能需要采取行动

这三种反馈结合，让 ClawPet 真正形成一个"感知 → 反馈 → 闭环"的完整交互。

## 6. 用 SOUL.md 驱动"灵魂模式"

即使 Gateway 这条线暂时没接上，或者你想给 agent 更明确的情绪/状态表达，也可以继续用 `SOUL.md + set_state.py` 这条桥接路线。

脚本在：

- `/Users/wuhao/clawpet/scripts/set_state.py`

例如：

```bash
python3 /Users/wuhao/clawpet/scripts/set_state.py coding "正在实现登录页"
python3 /Users/wuhao/clawpet/scripts/set_state.py thinking "在分析上下文"
python3 /Users/wuhao/clawpet/scripts/set_state.py running "正在跑测试"
python3 /Users/wuhao/clawpet/scripts/set_state.py idle "待命中"
```

推荐把下面这段加入你的 `SOUL.md`：

```md
## ClawPet 灵魂状态同步规则
- 接到任务时：先执行 `python3 /Users/wuhao/clawpet/scripts/set_state.py coding "<任务描述>"` 再开始工作
- 深度分析时：执行 `python3 /Users/wuhao/clawpet/scripts/set_state.py thinking "<正在思考的内容>"`
- 调工具或跑测试时：执行 `python3 /Users/wuhao/clawpet/scripts/set_state.py running "<当前动作>"`
- 等待反馈或被阻塞时：执行 `python3 /Users/wuhao/clawpet/scripts/set_state.py waiting "<等待原因>"`
- 完成任务后：执行 `python3 /Users/wuhao/clawpet/scripts/set_state.py idle "待命中"` 再回复
```

## 目前最推荐的日常使用姿势

如果你已经在用 OpenClaw，我建议这样玩：

1. 先启动 OpenClaw Gateway。
2. 再启动 ClawPet。
3. 平时让桌宠常驻在桌面边缘。
4. 需要时双击宠物，打开交接面板。
5. 收到审批浮窗时，直接在桌面上决策。
6. 看宠物在任务完成时欢呼雀跃、在失败时温和提示、在长时间等待时轻轻催促。
7. 想让 agent 更"有表情"时，再叠加 `SOUL.md` 状态同步规则。

这会形成一种很舒服的节奏：

- 平时它只是陪着你
- 忙起来时它会变得有存在感
- 需要你接手时它会明确把球传给你
- 完成时它会和你一起庆祝
- 失败时它会温和地提醒你
- 卡住时它会轻轻催促你

## 托盘菜单建议怎么用

托盘目前是主控制入口。

建议这样理解每个按钮：

- `添加蜜桃猫 / 薄荷猫 / 夜空猫`
  适合做多宠陪伴感，不同颜色混在桌面上会更有“办公室搭子”的感觉
- `给全部宠物换皮肤`
  适合快速切换桌面气质，现在还会记住你上次选的皮肤组合
- `打开 OpenClaw 面板`
  主工作入口
- `刷新 OpenClaw 状态`
  当你感觉状态不同步时手动拉一次
- `打开审批弹窗`
  适合你知道有审批但想强行拉起入口
- `灵魂模式`
  开着更像“和 agent 深度绑定”
- `点击穿透`
  适合你在长时间工作时不想让宠物拦鼠标
- `暂停动作`
  适合开会、录屏、演示时临时静止

## 现在最适合演示给别人看的场景

如果你想让别人 30 秒内理解这个项目，我推荐演示这几个动作：

1. 启动 ClawPet，让像素猫出现在桌面。
2. 双击宠物，打开 OpenClaw 面板。
3. 在面板里发一句话给当前 session。
4. 让宠物根据 Gateway 状态切换忙碌气质。
5. 触发一个审批请求，让审批浮窗弹出来。

这一套会很直观地说明：

这不是普通宠物，也不是普通控制台，它是一个“桌面化的 agent companion interface”。

## 你现在可以重点观察什么

如果你准备边用边提意见，我建议先看这几件事：

- 宠物在桌面上是否足够顺眼
- 托盘控制是不是够直接
- 面板是否足够轻，不像大后台
- 审批浮窗是不是够及时、够不打扰
- 宠物活动和 OpenClaw 状态之间的映射是否自然

## 当前已知边界

这版已经能玩，但还不是终版。

你现在看到的是一个非常能打的原型，仍然有这些边界：

- 宠物动画还是基于有限状态，不是完整骨骼或帧资源系统
- Gateway 活动类型已经细化，但还没有把每一种 tool 的视觉表现打磨到极致
- 面板还偏“控制器”，还可以继续做得更像“桌宠对话界面”
- 审批浮窗已经能用，但还可以补更多上下文，比如风险等级、命令结构、session 来源

## 现在的皮肤系统到什么程度了

## 性能优化说明

ClawPet 在处理 OpenClaw 组织回复时的高频 snapshot 更新做了以下多层激进优化来防止界面卡死和宠物抖动：

### 第 1 层：主进程节流 (index.ts)

1. **Panel 推送节流 (100ms)**：主进程对发送给 panel 窗口的 snapshot 进行节流，限制最高频率为 10 次/秒。即使 OpenClaw 频繁推送快照（50-100 次/秒），panel 也只会最多每秒收到 10 个更新。

2. **宠物窗口推送节流 (500ms)**：宠物窗口收到 snapshot 更新的频率进一步降低到最多 2 次/秒。宠物只需要显示状态图标，不需要实时的快照内容更新。这大幅减少了主进程中 `setBounds` 调用的频率，防止宠物窗口高频率抖动。

### 第 2 层：Panel 前端流式更新优化 (panel-app.ts)

3. **结构化变化检测**：面板应用区分两类变化：
   - **结构性变化**（必须立即渲染）：消息数量变化、运行ID变化、审批变化、会话变化、连接状态变化
   - **流式变化**（可以延迟渲染）：`liveTranscript.text` 的实时更新（这是 OpenClaw 组织回复时最频繁的变化）
   
   只有结构性变化才会触发完整 DOM 重建，流式文本更新会被延迟合并。

4. **用户滚动保护**：当用户正在滚动时，面板会完全暂停所有渲染操作，直到滚动停止 150ms 后才恢复更新。这确保：
   - 用户滚动不会被 DOM 更新中断
   - 滚动事件不会与 render 操作竞争主线程
   - 滚动流畅度优先于内容更新速度

5. **异步渲染批处理（requestAnimationFrame）**：所有渲染操作通过 `requestAnimationFrame` 进行，多个快速连续的渲染请求会被合并为一个，减少 DOM 操作次数。

6. **IME 输入法保护**：在用户进行中文输入法组合输入时，会延迟 UI 渲染以避免中断输入法编辑。

### 第 3 层：宠物管理器优化 (pet-manager.ts)

7. **宠物 Snapshot 缓存**：宠物管理器会缓存最新的 snapshot，只在 500ms 的时间间隔后才真正广播给所有宠物窗口，减少宠物窗口的位置更新调用。

### 优化效果

这三层优化结合起来，可以达到：
- ✅ **滚动流畅** - 用户滚动时面板完全不会被打断，即使 OpenClaw 在频繁推送数据
- ✅ **宠物稳定** - 宠物不会因为频繁的 snapshot 而高频率抖动
- ✅ **输入正常** - 前端拼音输入工作正常，不会被渲染中断
- ✅ **内容更新** - 关键结构性变化（新消息、状态改变）会及时显示
- ✅ **流式内容** - 流式文本（如 thinking 过程）会在滚动停止后更新

### 原理总结

核心思想是 **用户交互优先**：在用户主动滚动或输入时，暂停所有不必要的自动更新，只在用户交互停止后才进行更新。这样可以：
- 在高流量情况下（组织回复）保持 UI 响应性
- 不牺牲关键信息的及时性（消息到达、状态变化）
- 将流式内容的更新延迟到合适的时机
