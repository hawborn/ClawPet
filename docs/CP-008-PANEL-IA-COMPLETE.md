### 2. 具体改进内容

#### 2.1 简化头部卡片
```typescript
// 之前
<h1>OpenClaw</h1>
<p>{{runSummary}}</p>  // 冗长的状态文本
<span>{{sessionSubtitle}}</span>  // 额外信息

// 之后
<h1>Desktop Pet</h1>
<!-- 仅保留网关地址 -->
<span>{{gatewayUrl}}</span>
```

**移除内容**：
- `panel-summary-text`（活动状态文本）
- `sessionSubtitle`（会话信息）

**效果**：头部从"信息罗列"变成"清晰标识"

#### 2.2 强化任务卡片视觉优先级
```css

  border-left: 6px solid var(--leaf-deep);  /* 加粗边框 */
  box-shadow: 6px 6px 0 rgba(...), inset 0 1px 0 rgba(159, 193, 113, 0.3);  /* 增强阴影 */
  /* 背景对比度增加 */

}

/* 任务标签 */
.current-task-content .task-label {
  font-size: 14px;  /* 13px → 14px */
  font-weight: 700;  /* 600 → 700 */
}
```

**效果**：用户能一眼看到最重要的任务

#### 2.3 强化待接球区域
```css

  border-left: 6px solid #ffba6b;  /* 加粗边框 */

  background: linear-gradient(180deg, rgba(255, 186, 107, 0.08) 0%, ...);
}
```

**效果**：待审批/待决策的事项形成明确的视觉重点

#### 2.4 会话列表轻量化
```typescript

// 包含：session.kind + session.lastChannel + relativeTime(session.updatedAt)

// 之后

```

**变化**：
- 移除 `kind` 和 `lastChannel`
- 保留最后活动时间戳
- 会话列表从"信息卡"变成"快速开关"

#### 2.5 隐藏统计卡片
```typescript

- 会话统计 (N 个可见会话)
- 在线实例统计

- 审批数量
```

**理由**：这些数据对于"理解现在最重要的是什么"无帮助，只会产生"管理台感"

```css

  box-shadow: 4px 4px 0 rgba(90, 52, 31, 0.12);  /* 4px 而不是 6px */
```

**效果**：次要功能不会抢占视觉焦点

#### 2.7 对话区域的优化

## 🎨 设计原则应用

### 1. 信息优先级明确
```
最高优先级（强调）：
  → 当前最重要任务 🎯
  → 待我接球 ⚽

中等优先级（必需）：
  → 会话列表

低优先级（可选）：
  → 衣橱
```

### 2. 降低认知负荷
- **移除冗余信息**：统计数字、元数据
- **简化视觉**：每个卡片目的清晰
- **保留关键操作**：会话切换、皮肤选择

### 3. 工作流对齐
```
用户进入面板时的心智模型：
1. "现在最重要的任务是什么？" → 🎯 当前最重要任务
2. "我需要做什么？" → ⚽ 待接球
3. "我要在哪个会话工作？" → 🎯 切换会话
4. "我想换个皮肤吗？" → 👗 衣橱（可选）
```

## 🎓 设计决策

### 为什么移除统计卡片？
1. **用户心智模型**：用户进入面板是为了"了解当前状态和采取行动"，不是"查看系统统计"
2. **信息冗余**：会话数、实例数、节点数对决策没有帮助
3. **降低噪音**：移除这些卡片后，关键信息更突出

### 为什么保留会话列表？
1. **核心操作**：用户需要在会话间切换
2. **轻量实现**：简化后的会话列表体积小，易扫描
3. **工作流必需**：不同会话代表不同任务上下文

### 为什么强化任务卡片？
1. **用户期待**：进入面板第一眼应该看到"现在在做什么"
2. **视觉层级**：更粗的边框 + 更强的阴影 = 更明确的优先级
3. **信息价值**：每个卡片上的任务描述都很重要
