# 行为准则

## 我们的承诺

为了营造一个开放友好的环境，我们作为贡献者和维护者承诺：

- 尊重所有人，无论其性别、性别认同、性表达、种族、民族、年龄、能力、教育、社会经济地位或其他身份
- 创造一个包容性的环境
- 优先考虑项目和社区的最佳利益
- 对批评保持开放态度，同时给予和接受建设性反馈
- 专注于对社区最有利的事情

## 我们的标准

有利于创建积极环境的行为包括：

- 使用热情和包容的语言
- 尊重不同的观点和经历
- 接受建设性的批评
- 专注于对社区最有利的事情
- 向其他社区成员表示同情

不可接受的行为包括：

- 使用性化语言或意象以及不欢迎的性关注或进展
- 人身攻击、侮辱或贬低性评论
- 骚扰（线上或线下）
- 未经明确许可发布他人私人信息，例如住址或电子邮件地址
- 其他明显不专业或不受欢迎的行为

## 执行

项目维护人员负责明确可接受行为的标准，并应针对任何不可接受行为采取适当和公平的纠正措施。

项目维护人员有权利和责任删除、编辑或拒绝与本行为准则不符的评论、提交、代码、Wiki 编辑、问题和其他贡献。

## 范围

本行为准则适用于所有项目空间（GitHub issues、pull requests、discussions、代码、文档等）以及代表项目或其社区的任何公共空间（邮件列表、社交媒体、会议等）。

## 报告问题

如果你遇到了不可接受的行为，请通过以下方式报告：

- 直接向项目维护人员报告
- 在相关 issue 中添加报告
- 使用 GitHub 的报告功能

所有报告都将被认真对待和调查。

## 执行准则

项目维护人员将遵循这些执行准则：

1. **首次违规**：警告和更正
2. **持续违规**：暂时禁止参与项目讨论和贡献
3. **严重或持续**：永久禁止参与项目

## 致谢
